## Context

Every PR in the Pre-Iter-0 phase landed a typed entry under `## [Unreleased]` → `### Pre-Iter-0` with a consistent shape: a bolded "<Kind> (<branch-name>):" header followed by indented bullets. The pattern repeated 5+ times in this repo (Preflight-Infra, Preflight-Fix, Preflight-Fix-2, Iter-0 Launchpad, Slack-PR Hardening). Hand-editing is tedious and easy to skew (German vs English, bullet depth, finding-ID format).

This skill formalises the template.

## Your Task

Append a new sub-bullet block to `CHANGELOG.md` under the existing `### Pre-Iter-0` section, summarising what a freshly-merged PR added.

### Steps

1. Confirm scope. Ask user (one AskUserQuestion) for:
   - PR number that just merged (or "current branch" if not yet merged).
   - Kind label (free text, e.g. "Preflight-Fix", "Iter-0 Launchpad", "Slack-PR Hardening").
   - Branch name (default: derive from `gh pr view <N> --json headRefName`).

2. Pull the PR's commit subjects via `gh pr view <N> --json commits --jq '.commits[].messageHeadline'`. Each commit subject becomes a candidate bullet.

3. Read `CHANGELOG.md`. Locate the `### Pre-Iter-0` block.

4. Compose the new entry block:
   ```markdown
   - **<Kind> (<branch>):**
     - <First commit subject>: <one-line rationale>.
     - <Second commit subject>: <one-line rationale>.
     - ...
   ```
   For each commit subject, derive the rationale either from the commit body (preferred — first paragraph after the subject) or from the PR's `body` field. Keep each bullet to one line.

5. Append the block at the END of the `### Pre-Iter-0` block (after the last existing sub-bullet).

6. Show the diff to the user. Ask whether to commit + push directly or leave staged.

7. If approved: commit on the current branch with message `docs(changelog): <kind> entry under Pre-Iter-0` and push.

### Guardrails

- The current branch must NOT be `main`. If on main, refuse and tell the user to open a follow-up branch (CHANGELOG-only PRs go through reviewer-chain-attestation like any other).
- The CHANGELOG already has a header `## [Unreleased]` and `### Pre-Iter-0`. If either is missing, abort and tell the user the file shape changed.
- Language: match the existing entries (German bullets in this repo).
- Do not modify existing entries.
