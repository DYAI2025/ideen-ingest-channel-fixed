## Context

This repo tracks every time-bound technical-debt item as a GitHub issue with the `sunset` label and an `iter-N` milestone. The pattern showed up twice already: once for the 12 ADR sunsets (#11–#22), once for the 7 Iter-deferred findings from a code review (#24–#30). The bulk-loop is tedious to hand-roll each time.

This skill formalises the loop.

## Your Task

Open a batch of sunset-tracker issues, one per row of a user-provided TSV (or interactively-constructed list).

### Steps

1. Determine the input source. Ask user (one AskUserQuestion):
   - "Inline TSV (paste into chat) or read from a file path?"
   - Format: `<iter-milestone>\t<short-slug>\t<body-text>` per line. Example:
     ```
     iter-2	slack-c3-dedup-table	C3: slack_event_dedup table missing. Iter 2 Alembic adds it.
     ```

2. Validate every row:
   - Milestone exists: `gh api repos/<owner>/<repo>/milestones --jq '.[].title'`. If not, abort with the missing-milestone list — caller can fix and re-run.
   - Labels `sunset` and the iter label exist: `gh label list | grep -E "^(sunset|iter-N)\s"`. If not, abort.

3. For each row, construct title and body:
   - Title: `Sunset: <short-slug>` (matches the convention used in #11–#22).
   - Body: the third TSV column verbatim, followed by a footer:
     ```
     Tracked via [ADR-016](../docs/adr/ADR-016-governance-waiver-preflight-no-review.md) sunset convention.
     Closing this issue is a precondition for entering the named iter.
     ```

4. Loop:
   ```bash
   while IFS=$'\t' read -r MS SLUG BODY; do
     gh issue create \
       --title "Sunset: $SLUG" \
       --body "$BODY"$'\n\n'"<footer text>" \
       --label "sunset,$MS" \
       --milestone "$MS" \
       --assignee "$GH_USER"
   done < "$INPUT"
   ```
   `$GH_USER` defaults to the result of `gh api user --jq .login`.

5. After the loop, print a summary: count + URL list.

6. Verify: `gh issue list --label sunset --state open --limit 50 --json number,title,milestone --jq '.[] | "\(.number)\t\(.milestone.title)\t\(.title)"'` and show the appended rows.

### Guardrails

- Refuse to run if `gh auth status` reports unauthenticated.
- If a row's milestone doesn't exist, ABORT the entire batch (don't partially-create; leaves an inconsistent set). Tell the user which row failed.
- If the same slug already has an open issue (`gh issue list --label sunset --search "Sunset: <slug>"`), skip and log — don't open a duplicate.
- Do not pass the `--body` text through a shell expansion that could interpret backticks. Use single-quoted heredoc.
